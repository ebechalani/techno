Programmer : traiter une image

Transformer une image pixel par pixel en Python : niveaux de gris et négatif.

💾 Tes réponses sont enregistrées automatiquement sur cet appareil.

Une image, ce sont des nombres

Une image numérique est une grille de pixels. Chaque pixel a une couleur codée par trois nombres de 0 à 255 : la quantité de Rouge, de Vert et de Bleu (codage RVB). En Python, on peut représenter un pixel par un triplet : (r, v, b).

Transformer une image, c'est appliquer un calcul à chaque pixel.

Activité 1 — Passer en niveaux de gris

Pour obtenir un pixel gris, on remplace R, V et B par leur moyenne. Un pixel gris a ses trois composantes égales.

def niveau_de_gris(pixel):
    r, v, b = pixel
    moyenne = (r + v + b) // 3
    return (moyenne, moyenne, moyenne)

print(niveau_de_gris((100, 150, 200)))

a. Calcule la moyenne de (100, 150, 200), puis donne le pixel gris obtenu.

b. Pourquoi les trois composantes d'un pixel gris sont-elles identiques ?

Activité 2 — Le négatif d'une image

Pour obtenir le négatif, on remplace chaque composante par 255 moins sa valeur.

def negatif(pixel):
    r, v, b = pixel
    return (255 - r, 255 - v, 255 - b)

print(negatif((0, 0, 0)))
print(negatif((255, 0, 0)))

a. Quel pixel obtient-on avec negatif((0, 0, 0)) ? Quelle couleur est-ce ?

b. Le négatif du rouge pur (255, 0, 0) donne quel triplet ?

Activité 3 — Traiter toute l'image

Pour traiter toute l'image, on applique la fonction à chaque pixel, à l'aide d'une boucle. C'est le principe des filtres des logiciels de retouche.

Quel outil de programmation permet de répéter le même calcul sur tous les pixels ?

Ce que je retiens

Une image est une grille de pixels RVB, chaque composante allant de 0 à 255. On traite une image en Python en appliquant un calcul à chaque pixel : la moyenne des composantes donne un niveau de gris, et 255 − valeur donne le négatif. Une boucle permet d'appliquer le filtre à toute l'image.
