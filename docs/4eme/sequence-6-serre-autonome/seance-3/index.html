Séance 3 : Programmer le système

De l'algorithme au programme : rendre la serre autonome.

🎯
Objectifs
  • Identifier les entrées (capteurs) et les sorties (actionneurs) du système.
  • Écrire l'algorithme d'une fonction automatique.
  • Traduire un algorithme en programme (repère de 4ᵉ).
💾 Tes réponses sont enregistrées automatiquement sur cet appareil.

Où en est le projet ?

Notre serre doit fonctionner toute seule : arroser quand la terre est sèche, éclairer quand il fait sombre, aérer quand il fait trop chaud. Pour cela, une carte programmable (micro:bit ou Arduino) lit les capteurs et commande les actionneurs.

Activité 4 — De l'algorithme au programme

Pour que la serre réagisse en permanence, tout le programme est placé dans une boucle « répéter indéfiniment ». Chaque test « Si … Alors … Sinon … » devient un bloc conditionnel dans le logiciel de programmation (MakeCode pour micro:bit, ou mBlock/Arduino).

À retenir dans ton programme :

  • une variable stocke la valeur lue par un capteur (ex. humidite) ;
  • une condition compare cette variable à un seuil (humidite < 30) ;
  • une boucle répète les tests en continu.

Quel bloc utilise-t-on pour comparer la valeur d'un capteur à un seuil ?

Ce que je retiens

Un système automatique fonctionne selon le schéma capteur → programme → actionneur. Programmer, c'est d'abord écrire un algorithme (Si… Alors… Sinon…), puis le traduire en blocs dans une boucle qui se répète, pour que le système réagisse tout seul à son environnement.
